## 5.8 Cómo configurar motivos de anulación en el Panel Administrativo – YURES

El módulo **Justificación de anulaciones** permite definir los mensajes obligatorios que el sistema mostrará cuando un usuario anule un pedido, documento o transacción.

Configurar correctamente estos motivos es fundamental para:

* Mantener control interno.
* Evitar anulaciones sin sustento.
* Generar reportes auditables.
* Identificar errores operativos frecuentes.
* Mejorar la trazabilidad del negocio.

---

# 📍 Ruta de acceso

Desde el menú lateral:

**Catálogo y Reglas Operativas → Justificación de anulaciones**

Al ingresar visualizará:

* Botón **+ Registrar Mensaje**
* Tabla de motivos existentes
* Fecha y hora de registro
* Opciones de edición y eliminación
* Buscador
* Selector de cantidad de registros visibles

---

# 1️⃣ Registrar un nuevo motivo de anulación

## ➕ Paso a paso

1. Haga clic en el botón **+ Registrar Mensaje**.
2. Se abrirá una ventana emergente llamada **Mensajes de Anulación**.
3. En el campo **Descripción**, escriba el motivo.
4. Presione **GUARDAR**.

Ejemplos recomendados:

* Error en el registro del pedido
* Pedido duplicado
* Cliente canceló el pedido
* Producto no disponible
* Error de digitación
* Error en el precio

Después de guardar, el motivo aparecerá inmediatamente en la lista.

---

# 2️⃣ Buscar un motivo existente

En la parte superior derecha encontrará el campo **Buscar**.

1. Escriba una palabra clave.
2. El sistema filtrará automáticamente los resultados.

Ejemplo:
Escribir "error" mostrará todos los motivos que contengan esa palabra.

Esto facilita la revisión cuando existen muchos registros.

---

# 3️⃣ Modificar un motivo

Para editar un motivo existente:

1. Ubique el registro en la tabla.
2. En la columna **Acciones**, seleccione el desplegable.
3. Elija **Modificar datos**.
4. Se abrirá nuevamente la ventana emergente.
5. Edite la descripción.
6. Presione **GUARDAR**.

Los cambios se reflejarán inmediatamente.

---

# 4️⃣ Eliminar un motivo

Si necesita eliminar un motivo:

1. En la columna **Acciones**, seleccione el desplegable.

2. Haga clic en **Eliminar registro**.

3. El sistema mostrará el mensaje de confirmación:

   > ¿Desea eliminar este registro?

4. Presione **Aceptar** para confirmar.

5. O **Cancelar** para mantenerlo.

⚠ Recomendación importante:
No elimine motivos que ya hayan sido utilizados en anulaciones registradas, ya que pueden afectar la coherencia histórica de reportes.

---

# 5️⃣ Cómo funcionan los motivos en el proceso de venta

Cuando un usuario intenta anular:

* Un pedido
* Un documento
* Una venta
* Una operación

El sistema obligará a seleccionar uno de los motivos configurados.

Esto garantiza que cada anulación tenga respaldo.

Ejemplo práctico:

Un cajero anula una venta porque ingresó mal el producto.
El sistema solicitará elegir un motivo.
Selecciona: **Error de digitación**.

El sistema guarda:

* Usuario
* Fecha
* Hora
* Motivo seleccionado

Esta información puede utilizarse posteriormente en auditorías y reportes.

---

# 6️⃣ Importancia para control interno

Configurar adecuadamente los motivos permite:

✔ Detectar problemas recurrentes
✔ Medir errores del personal
✔ Evaluar fallas en inventario
✔ Identificar errores de precios
✔ Mejorar capacitación del equipo

Ejemplo de análisis:

Si el 40% de anulaciones se debe a "Error en el precio", existe un problema en la configuración del catálogo.

---

# 7️⃣ Buenas prácticas recomendadas

✔ Mantener los motivos claros y específicos.
✔ No crear descripciones ambiguas como "Otro".
✔ Revisar periódicamente la lista.
✔ Limitar el número de motivos para evitar confusión.
✔ Estandarizar redacción (usar frases cortas y directas).

---

# 8️⃣ Casos comunes de uso por tipo de negocio

### Restaurante

* Cliente cambia de opinión
* Producto agotado
* Error de digitación
* Pedido duplicado

### Tienda retail

* Producto mal escaneado
* Precio incorrecto
* Cliente desistió

### Delivery

* Pedido cancelado por cliente
* Dirección incorrecta
* Tiempo excedido

---

# 🎯 Resultado esperado

Al finalizar la configuración:

* Cada anulación tendrá una justificación obligatoria.
* Se reducirá el uso de anulaciones informales.
* Mejorará la trazabilidad del sistema.
* Se fortalecerá el control administrativo.

---

La correcta configuración de los motivos de anulación en YURES no solo cumple una función operativa, sino también estratégica, ya que permite convertir las anulaciones en información útil para la mejora continua del negocio.
