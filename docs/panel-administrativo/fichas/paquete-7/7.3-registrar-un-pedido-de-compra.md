# 7.3 Cómo registrar un pedido de compra en el Panel Administrativo – YURES

El **Pedido de Compra** es un documento interno que se utiliza para solicitar productos a un proveedor antes de emitir una orden o registrar una compra definitiva.

A diferencia de la Orden de Compra, el Pedido permite:

* Planificar abastecimiento.
* Controlar requerimientos pendientes.
* Gestionar aprobaciones.
* Separar solicitud de compra y registro contable.
* Controlar estados (Pendiente / Aprobado).

---

## 📍 Ruta de acceso

Menú lateral:

**Compras y Proveedores → Pedidos de compra**

---

# 1️⃣ Ingresar al módulo de Pedidos de Compra

Al ingresar verá:

* Botón **+ Registrar pedido de compra**
* Buscador
* Lista de pedidos registrados
* Estado del pedido (Pendiente / Aprobado)
* Opciones disponibles

Para crear uno nuevo:

Presione **+ Registrar pedido de compra**

---

# 2️⃣ Pantalla de Registro de Compras (Pedido)

Se abrirá el formulario donde deberá completar los datos del proveedor y los productos solicitados.

---

# 3️⃣ Registrar datos del proveedor

## ✔ Documento Proveedor *

Ingrese el DNI o RUC del proveedor.

Si el proveedor ya existe, el sistema puede autocompletar.

Si no existe:

Presione **+ Nuevo Proveedor**

Se abrirá una ventana emergente para registrarlo sin salir del pedido.

---

## ✔ Registrar nuevo proveedor (si no existe)

Complete:

* Tipo de documento
* Nº Documento
* Nombre o razón social
* Nº celular o teléfono
* Dirección
* Referencia
* Email
* Observación

Presione **Guardar**.

El proveedor quedará disponible para el pedido.

---

# 4️⃣ Datos generales del pedido

## ✔ Almacén destino *

Seleccione el almacén donde se espera que ingresen los productos.

Ejemplo:

* Almacén General
* Almacén de Ventas

Este campo es importante porque define el destino del stock cuando se apruebe o convierta en compra.

---

## ✔ Fecha de vencimiento

Se usa cuando el pedido contempla condiciones de crédito.

---

## ✔ Observación

Puede registrar:

* Motivo del pedido
* Área solicitante
* Notas internas
* Referencia de requerimiento

---

# 5️⃣ Agregar productos al pedido

En la sección **Descripción**:

1. Seleccione el producto.
2. Presione **Agregar**.

El producto aparecerá en la tabla inferior.

---

## ✔ Información por cada producto

Para cada ítem debe registrar:

* Cantidad
* Unidad
* Presentación
* Precio proveedor

El sistema calculará automáticamente el costo total por producto y el total general del pedido.

---

# 6️⃣ Crear producto nuevo desde el pedido

Si el producto no existe:

Presione **+ Nuevo Producto**

Se abrirá una ventana con dos pestañas:

---

## 🟦 Información General

Complete:

* Nombre del producto (obligatorio)
* Código (opcional)
* Tipo de afectación IGV
* Categoría
* Precio
* Impresora (si aplica)
* Descripción

Presione **Guardar**.

---

## 🟦 Adicionales

Configure:

* Tipo (Ingrediente, Platillo, Industrializado, Producido)
* Compra / Venta
* Unidad
* Presentación
* Costo
* Kardex (Sí / No)
* Precio modificable
* Combo
* Límite unidad
* Cantidad
* Tiempo de preparación
* Cuenta contable

Presione **Guardar** para finalizar.

El producto quedará disponible para agregar al pedido.

---

# 7️⃣ Eliminar productos del pedido

Si agregó un producto incorrectamente:

Presione el botón **Eliminar** en la fila correspondiente.

El sistema mostrará una confirmación:

“¿Realmente desea eliminar esta descripción de la compra?”

Presione:

* **Aceptar** para confirmar
* **Cancelar** para mantenerlo

---

# 8️⃣ Revisar el total del pedido

En la parte inferior visualizará:

* Total acumulado del pedido

Es recomendable verificar cantidades y precios antes de guardar.

---

# 9️⃣ Guardar el pedido

Una vez completados todos los datos:

Presione **Guardar** (parte superior derecha).

El pedido quedará registrado.

---

# 🔟 Estados del pedido

En la lista principal podrá ver:

* Estado: Pendiente
* Estado: Aprobado

Estos estados permiten controlar el flujo interno antes de convertirlo en compra definitiva.

---

# 1️⃣1️⃣ Consultar pedidos registrados

Desde la lista podrá:

* Buscar por proveedor.
* Filtrar por fecha.
* Revisar totales.
* Consultar estado.
* Gestionar aprobación.

---

# 1️⃣2️⃣ Diferencia entre Pedido y Orden de Compra

| Pedido de Compra               | Orden de Compra                    |
| ------------------------------ | ---------------------------------- |
| Documento interno de solicitud | Documento formal de compra         |
| Puede estar pendiente          | Impacta directamente el stock      |
| Permite control previo         | Registra ingreso real de productos |
| Se usa para planificación      | Se usa para registro contable      |

---

# 1️⃣3️⃣ Buenas prácticas

✔ Registrar pedidos antes de comprar grandes volúmenes.
✔ Validar proveedor antes de aprobar.
✔ Confirmar precios negociados.
✔ No duplicar pedidos.
✔ Revisar cantidades antes de guardar.
✔ Usar observaciones para trazabilidad interna.

---

# 1️⃣4️⃣ Errores comunes a evitar

* No seleccionar almacén correcto.
* Registrar productos duplicados.
* No validar precios.
* Aprobar sin revisión.
* Olvidar guardar cambios.

---

# 🎯 Resultado esperado

Al finalizar correctamente el registro:

* El pedido queda documentado.
* Se mantiene control del abastecimiento.
* Se puede aprobar o convertir en compra.
* Se mejora la planificación financiera.
* Se evita desorden en adquisiciones.

---

El módulo **Pedidos de Compra** permite estructurar el proceso de abastecimiento antes de afectar inventario o contabilidad, brindando mayor control y organización al negocio.
