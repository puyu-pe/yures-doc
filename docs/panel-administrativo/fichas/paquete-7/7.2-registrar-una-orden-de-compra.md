# 7.2 Cómo registrar una orden de compra en el Panel Administrativo – YURES

La **Orden de Compra** es el documento interno que permite registrar formalmente una compra de productos a un proveedor antes o durante su ingreso al almacén.
Este módulo permite llevar control documental, validar costos y organizar el abastecimiento del negocio.

Registrar correctamente una orden de compra garantiza:

* Control del inventario.
* Registro correcto de costos.
* Trazabilidad de productos.
* Orden administrativo.
* Sustento contable.
* Control de proveedores.

---

## 📍 Ruta de acceso

Menú lateral:

**Compras y Proveedores → Órdenes de compra**

---

# 1️⃣ Ingresar al módulo de Órdenes de compra

Al ingresar visualizará:

* Botón **+ Registrar Compra**
* Buscador
* Lista de órdenes registradas
* Botón **Ver** para revisar cada orden

Si desea crear una nueva orden:

Presione **+ Registrar Compra**

---

# 2️⃣ Pantalla de Registro de Compras

Se abrirá el formulario **Registro de Compras**.

En esta sección se debe completar información obligatoria y opcional.

---

# 3️⃣ Completar datos del proveedor

## ✔ Documento Proveedor *

Ingrese el número de DNI o RUC del proveedor.

El sistema puede autocompletar si el proveedor ya existe.

Si el proveedor no está registrado:

Presione **+ Nuevo Proveedor**

Se abrirá el formulario para registrarlo sin salir de la orden.

---

## ✔ Nombre proveedor / Razón Social *

Debe coincidir con el documento ingresado.

Es obligatorio para validar la compra.

---

# 4️⃣ Datos del documento de compra

## ✔ Tipo de documento *

Seleccione:

* Factura
* Boleta
* Otro tipo habilitado

Esto depende del comprobante entregado por el proveedor.

---

## ✔ Serie *

Ingrese la serie del comprobante.

Ejemplo:
F001
B001

---

## ✔ Numeración *

Ingrese el número correlativo del comprobante.

Ejemplo:
125
3580

Debe coincidir exactamente con el documento físico o electrónico.

---

## ✔ Fecha de Emisión *

Fecha del comprobante del proveedor.

---

## ✔ Fecha de vencimiento

Opcional.
Se usa cuando la compra es al crédito.

---

## ✔ Fecha de Registro *

Fecha en la que se está registrando en el sistema.

Puede coincidir o no con la fecha de emisión.

---

## ✔ Almacén destino *

Seleccione el almacén donde ingresará el producto:

* Almacén General
* Almacén de Ventas
* Otro almacén configurado

Es fundamental elegir correctamente, ya que impactará el stock.

---

## ✔ Observación

Puede registrar:

* Condiciones de pago
* Número de guía
* Referencia de pedido
* Notas internas

---

# 5️⃣ Agregar productos a la orden

En la sección **Descripción**:

1. Seleccione el producto desde el desplegable.
2. Presione **Agregar**.

El producto aparecerá en la tabla inferior.

---

## ✔ Campos por producto

Para cada producto debe completar:

* Cantidad
* Precio proveedor
* Unidad
* Presentación

El sistema calculará automáticamente:

* Subtotal
* IGV
* Total

---

# 6️⃣ Crear producto nuevo desde la orden

Si el producto no existe:

Presione **+ Nuevo Producto**

Se abrirá una ventana emergente con dos pestañas:

### Información General

Complete:

* Nombre del producto (obligatorio)
* Código (opcional)
* Tipo de afectación IGV
* Categoría
* Precio
* Impresora (si aplica)
* Descripción

Presione **Guardar**.

---

### Adicionales

Configure:

* Tipo (Ingrediente, Platillo, Industrializado, etc.)
* Compra / Venta
* Unidad
* Presentación
* Costo
* Kardex
* Precio modificable
* Límite de unidad
* Cantidad
* Cuenta contable

Presione **Guardar** para finalizar.

El producto quedará disponible para agregar a la orden.

---

# 7️⃣ Eliminar un producto agregado

Si se agregó un producto incorrecto:

Presione el botón **Eliminar** (rojo) en la fila correspondiente.

El sistema solicitará confirmación:

“¿Realmente desea eliminar esta descripción de la compra?”

Presione:

* **Aceptar** para confirmar
* **Cancelar** para mantenerlo

---

# 8️⃣ Ver totales automáticos

En la parte inferior verá:

* Subtotal
* IGV
* Percepción (si aplica)
* Total final

Estos valores se calculan automáticamente según precios ingresados.

Es importante revisar antes de guardar.

---

# 9️⃣ Guardar la orden de compra

Una vez completada la información:

Presione **Guardar** (parte superior derecha).

La orden quedará registrada y:

* Se actualizará el stock.
* Se guardará el historial.
* Se podrá consultar desde la lista.

---

# 🔟 Ver lista de órdenes registradas

Presione **Ver Lista** para regresar al listado general.

Ahí podrá:

* Consultar órdenes registradas.
* Buscar por proveedor.
* Filtrar por fecha.
* Revisar totales.
* Visualizar detalles.

---

# 1️⃣1️⃣ Buenas prácticas al registrar órdenes

✔ Verificar que el proveedor esté correctamente registrado.
✔ Validar serie y numeración antes de guardar.
✔ Confirmar almacén destino correcto.
✔ Revisar precios antes de confirmar.
✔ No duplicar órdenes.
✔ Registrar inmediatamente después de recibir el comprobante.

---

# 1️⃣2️⃣ Errores comunes a evitar

* Registrar en almacén incorrecto.
* No ingresar correctamente la serie.
* Registrar precios erróneos.
* Olvidar agregar productos.
* Eliminar productos sin confirmar.
* Registrar compras duplicadas.

---

# 1️⃣3️⃣ Impacto en el sistema

Una orden de compra correctamente registrada:

* Aumenta el stock del almacén seleccionado.
* Actualiza el Kardex.
* Alimenta reportes de compras.
* Impacta el control de costos.
* Sirve como respaldo contable.

---

# 🎯 Resultado esperado

Al finalizar el proceso:

* La orden queda documentada.
* El inventario se actualiza automáticamente.
* El historial de compras queda registrado.
* El negocio mantiene control financiero y operativo.

---

La correcta gestión de órdenes de compra es clave para mantener el equilibrio entre inventario, costos y rentabilidad del negocio.
