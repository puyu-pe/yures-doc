# 7.6 Cómo registrar una orden de compra desde un pedido en el Panel Administrativo – YURES

Registrar una **orden de compra** desde un pedido aprobado permite formalizar la compra con el proveedor, generar el documento correspondiente (factura, boleta u otro) y registrar el impacto contable y de inventario en el sistema.

Este proceso convierte un **requerimiento interno (pedido de compra)** en una **compra real registrada**, vinculando proveedor, documento, almacén y productos.

---

## 📍 Ruta de acceso

Menú lateral:

**Compras y Proveedores → Órdenes de compra**

---

# 1️⃣ Ingresar al módulo Órdenes de compra

Al ingresar visualizará el listado de compras ya registradas.
En la parte superior encontrará el botón:

**+ Registrar Compra**

También puede visualizar compras anteriores mediante el botón **Ver** en la columna Opciones.

---

# 2️⃣ Registrar una nueva compra

Haga clic en:

**+ Registrar Compra**

Se abrirá el formulario **Registro de compras**.

---

# 3️⃣ Completar los datos del proveedor

### ✔ Documento Proveedor (obligatorio)

Ingrese el número de RUC o documento del proveedor.

### ✔ Nombre proveedor / Razón Social (obligatorio)

El sistema puede autocompletar si el proveedor ya existe.

Si el proveedor no está registrado:

* Haga clic en **+ Nuevo Proveedor**
* O use **Editar Proveedor** si necesita actualizar datos.

Es fundamental validar que:

* El RUC sea correcto.
* La razón social coincida.
* El proveedor esté activo.

---

# 4️⃣ Datos del documento de compra

### ✔ Tipo de documento

Seleccione el tipo de comprobante:

* Factura
* Boleta
* Otro documento autorizado

### ✔ Serie

Ejemplo: F001

### ✔ Numeración

Ejemplo: 15

Estos datos deben coincidir exactamente con el comprobante físico o electrónico del proveedor.

---

# 5️⃣ Fechas importantes

### ✔ Fecha Emisión

Fecha del comprobante emitido por el proveedor.

### ✔ Fecha de Registro

Fecha en que se registra la compra en el sistema.

### ✔ Fecha de vencimiento

Si aplica crédito o pago diferido.

Es importante que las fechas coincidan con la realidad contable.

---

# 6️⃣ Validar vinculación con el pedido

En el formulario se muestra la información del requerimiento original:

* Tipo de documento: Pedido de compra
* Serie Req
* Numeración Req

Esto permite mantener trazabilidad entre:

**Pedido → Orden de compra → Compra registrada**

---

# 7️⃣ Verificar productos y cantidades

En la sección inferior se muestran los productos del pedido:

* Descripción
* Cantidad
* Unidad
* Presentación
* Precio proveedor
* Costo total

Debe validar:

✔ Que la cantidad coincida con lo recibido.
✔ Que el precio sea el acordado.
✔ Que el subtotal y total sean correctos.

---

# 8️⃣ Validar cálculos

El sistema mostrará:

* Subtotal
* IGV
* Percepción (si aplica)
* Total

Antes de guardar:

* Confirme que el total coincida con el comprobante.
* Revise que el IGV esté correctamente calculado.
* Verifique que no existan errores en montos.

Un error en esta etapa afectará:

* Costos de inventario.
* Estados financieros.
* Margen de utilidad.
* Reportes contables.

---

# 9️⃣ Guardar la orden de compra

Una vez validado todo:

Proceda a guardar el registro.

El sistema:

* Genera la orden de compra.
* Actualiza inventario.
* Registra movimiento contable.
* Mantiene vinculación con el pedido original.

---

# 🔟 Confirmar registro

Regrese a:

**Compras y Proveedores → Órdenes de compra**

Verifique que el documento aparezca en el listado.

Puede hacer clic en **Ver** para revisar el detalle.

---

# 📊 Diferencia entre Pedido y Orden de compra

| Pedido de compra              | Orden de compra          |
| ----------------------------- | ------------------------ |
| Requerimiento interno         | Compra formal registrada |
| No afecta inventario          | Sí afecta inventario     |
| No genera obligación contable | Sí genera obligación     |
| Puede estar pendiente         | Documento definitivo     |

---

# ⚠ Consideraciones importantes

* No registrar una compra sin pedido aprobado.
* No alterar cantidades sin justificación.
* No registrar documento duplicado.
* Verificar que el almacén destino sea correcto.
* Confirmar que el proveedor sea válido.

---

# 🛑 Errores comunes

* Registrar compra sin validar precios.
* Confundir fecha de emisión con fecha de registro.
* Ingresar mal la serie.
* No verificar IGV.
* No revisar el total antes de guardar.

---

# 🎯 Buenas prácticas administrativas

✔ Validar físicamente el comprobante antes de registrar.
✔ Revisar el pedido aprobado antes de convertirlo.
✔ Confirmar recepción real de productos.
✔ Registrar inmediatamente después de recibir mercancía.
✔ Mantener coherencia entre pedido, orden y factura.

---

# 📌 Impacto del registro correcto

Cuando el proceso se realiza correctamente:

* El inventario se actualiza adecuadamente.
* El costo promedio se mantiene correcto.
* Se mejora el control financiero.
* Se mantiene trazabilidad completa.
* Se evitan diferencias contables.

---

# 🎯 Resultado esperado

Al registrar correctamente una orden de compra desde un pedido:

* El requerimiento interno queda formalizado.
* La compra queda documentada.
* El inventario refleja la entrada real.
* El sistema mantiene coherencia administrativa.
* Se garantiza control y transparencia en el proceso de adquisiciones.

---

El módulo de **Órdenes de compra** en YURES es el punto clave donde el pedido interno se convierte en una operación financiera real dentro del sistema.
