# 8.5 Cómo configurar tipos de transacciones en el Panel Administrativo – YURES

El módulo **Tipos de transacciones** permite definir y gestionar las categorías que se utilizarán para clasificar los movimientos operativos, principalmente los gastos, dentro del sistema.

Estas configuraciones influyen directamente en:

* La organización de gastos operativos.
* La clasificación interna de egresos.
* La generación de reportes.
* El análisis financiero.
* El control administrativo diario.

Una correcta configuración evita desorden contable y facilita la toma de decisiones.

---

# 📍 Paso 1: Ingresar al módulo

Desde el menú lateral:

**Configuración del sistema → Tipos de transacciones**

Se mostrará la pantalla de **Clasificación para gastos operativos**, donde podrá visualizar:

* Tipo (Ingreso / Egreso)
* Descripción
* Observación
* Opciones (Editar / Eliminar)

---

# ➕ Paso 2: Registrar un nuevo tipo de transacción

Haga clic en:

**+ Registrar**

Se abrirá el formulario **Registro de Tipo de Gasto**.

---

## 🧾 Campos del formulario

### 1️⃣ Descripción

Ingrese el nombre del tipo de transacción.

Ejemplos:

* Gastos del día
* Transporte
* Adelantos
* Consumos
* Pagos del día
* Devolución local

La descripción debe ser clara y representar correctamente el tipo de movimiento que se desea clasificar.

---

### 2️⃣ Observación

Campo opcional.

Permite añadir información adicional para contextualizar el uso del tipo de transacción.

Ejemplo:

* Gastos que se hacen diariamente.
* Pago adelantado del personal.
* Movilidad operativa.
* Gastos prepagados.

Esto ayuda a mantener coherencia cuando hay varios usuarios registrando movimientos.

---

### 3️⃣ Tipo de Transacción

Seleccione si corresponde a:

* **Ingreso**
* **Egreso**

⚠ Es importante seleccionar correctamente el tipo, ya que esto determina cómo se comportará el movimiento dentro del sistema financiero.

---

## 💾 Paso 3: Guardar el tipo de transacción

Haga clic en:

**Guardar**

El nuevo tipo quedará disponible para su uso inmediato dentro de los módulos correspondientes.

---

# ✏ Paso 4: Editar un tipo de transacción

En la columna **Opciones**, haga clic en el ícono de edición.

Podrá:

* Modificar la descripción.
* Ajustar la observación.
* Cambiar el tipo si es necesario.

Luego presione **Guardar** para confirmar.

⚠ Evite modificar tipos que ya estén asociados a muchos movimientos históricos sin revisar el impacto administrativo.

---

# 🗑 Paso 5: Eliminar un tipo de transacción

En la columna **Opciones**, haga clic en el ícono de eliminar.

El sistema mostrará un mensaje de confirmación:

¿Realmente desea eliminar este registro?

Seleccione:

* **Aceptar** para eliminar.
* **Cancelar** para mantener el registro.

---

## ⚠ Recomendaciones antes de eliminar

* Verifique que el tipo no esté siendo usado actualmente.
* No elimine tipos que tengan historial importante.
* Revise si está asociado a reportes contables.
* Consulte con el responsable administrativo si es necesario.

Eliminar un tipo usado puede generar inconsistencias en reportes históricos.

---

# 📊 Cómo se utilizan los tipos de transacciones

Los tipos de transacciones se usan para:

* Clasificar gastos operativos.
* Organizar reportes financieros.
* Diferenciar tipos de egresos.
* Facilitar análisis mensual.
* Estandarizar registros entre usuarios.

Ejemplo práctico:

En lugar de registrar gastos con conceptos libres sin estructura, el sistema permite clasificarlos bajo categorías predefinidas, manteniendo orden y consistencia.

---

# 🧠 Buenas prácticas

* Definir una estructura clara desde el inicio.
* Evitar crear demasiados tipos innecesarios.
* No duplicar categorías similares.
* Usar descripciones claras y estandarizadas.
* Revisar periódicamente la lista.
* Coordinar con el área contable.

---

# 🛑 Errores comunes

* Crear tipos ambiguos.
* No definir correctamente si es ingreso o egreso.
* Eliminar categorías activas.
* No usar observaciones explicativas.
* Permitir que cada usuario cree categorías sin control.

---

# 📈 Importancia administrativa

Un sistema sin clasificación estructurada de transacciones:

* Dificulta el análisis financiero.
* Genera reportes confusos.
* Reduce la trazabilidad.
* Complica auditorías.

Con una correcta configuración:

* Los gastos operativos se organizan adecuadamente.
* Se mejora el control interno.
* Se facilita el análisis mensual.
* Se mantiene coherencia administrativa.

---

# 🎯 Resultado esperado

Al configurar correctamente los tipos de transacciones:

* Los movimientos quedan correctamente clasificados.
* Se mejora el control financiero.
* Los reportes son más claros.
* Se facilita la supervisión administrativa.
* Se fortalece la estructura operativa del sistema.

---

El módulo **Tipos de transacciones** es una herramienta clave para mantener orden y coherencia en la gestión financiera operativa dentro del Panel Administrativo de YURES.
