# 5.5 Anular desde operativo: cuándo aplica y por qué requiere control

## Para qué sirve

Anular una venta realizada en el POS cuando se detecta un error el mismo día de emisión.

---

## Checklist rápido

* Documento correcto localizado.
* Venta emitida **el mismo día**.
* Motivo de anulación claro.
* Usuario con permiso o autorización de supervisor.

---

## Cuándo aplica

* Se registró una venta por error.
* El cliente canceló la compra inmediatamente.
* Se generó el documento incorrecto.
* La venta debe eliminarse antes del cierre operativo del día.

---

## Pasos

1. Presionar **Búsqueda venta**.
2. Buscar el documento por **tipo, serie y correlativo**.
3. Verificar que el documento mostrado sea el correcto.
4. Presionar **Anular**.
5. Escribir la **justificación de la anulación**.
6. Presionar **Anular** para confirmar.
7. Confirmar el mensaje **“Se anuló con éxito”**.

---

## Resultado esperado

* El documento queda anulado.
* El sistema registra la justificación.
* La venta deja de considerarse válida en la operación.

---

## Errores comunes

**Intentar anular un documento de otro día**
→ El sistema no lo permite.

**Anular el documento equivocado**
→ Siempre verificar el número antes de confirmar.

**No ingresar justificación**
→ El sistema exige indicar el motivo.

**Usuario sin permisos**
→ Puede requerir autorización de supervisor.
