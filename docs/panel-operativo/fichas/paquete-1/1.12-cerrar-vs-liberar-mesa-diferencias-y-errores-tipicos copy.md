# 1.11 Cerrar vs Liberar mesa: diferencias y errores típicos

Esta es una de las acciones que más afecta tu caja.
Confundirlas genera descuadres.

---

## 1️⃣ CERRAR MESA (proceso correcto)

### Cuándo usarlo

* Cliente paga.
* Se registra el pago.
* Se emite ticket / boleta / factura.
* La venta debe ingresar a caja.

### Qué hace el sistema

* Registra ingreso en caja.
* Marca la venta como pagada.
* Cambia la mesa a disponible.
* Guarda el movimiento para reportes.

### Flujo correcto

1. Ir a **CAJA**.
2. Liquidar venta.
3. Registrar pago.
4. Guardar.
5. Mesa queda libre correctamente.

---

## 2️⃣ LIBERAR MESA (acción administrativa)

### Cuándo usarlo

* Cliente se retiró sin pagar.
* Error operativo.
* Prueba de sistema.
* Cancelación total justificada.

### Qué hace el sistema

* No registra ingreso.
* No genera comprobante.
* Libera la mesa.
* Exige ingresar una razón.

Ejemplo:

> “EL CLIENTE SE RETIRÓ DEL LOCAL”

---

## Diferencia crítica

| Acción  | Registra ingreso | Afecta caja | Genera comprobante |
| ------- | ---------------- | ----------- | ------------------ |
| Cerrar  | Sí               | Sí          | Sí                 |
| Liberar | No               | No          | No                 |

---

## Error típico 1

Liberar mesa cuando el cliente sí pagó.

Consecuencia:

* Venta no entra a caja.
* Cuadre negativo.
* Reporte inconsistente.

---

## Error típico 2

Cerrar sin verificar monto pagado.

Consecuencia:

* Vuelto incorrecto.
* Diferencia en caja.

---

## Señal visual importante

* Mesa en rojo → activa.
* Mesa en verde → disponible.
* Después de cerrar correctamente, la mesa queda libre y el ingreso queda registrado.

---

## Regla operativa clara

Si hubo dinero → **Cerrar**
Si no hubo dinero → **Liberar (con justificación)**
