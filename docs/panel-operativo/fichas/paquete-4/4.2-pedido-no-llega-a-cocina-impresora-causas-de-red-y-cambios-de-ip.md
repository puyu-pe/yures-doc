# 4.2 Pedido no llega a cocina (impresora): causas de red y cambios de IP

## Para qué sirve

Identificar y resolver cuando los pedidos **sí se enviaron desde el POS pero no se imprimen en la cocina** debido a problemas de red o cambio de IP del equipo que maneja la impresión.

---

## Checklist rápido

* Pedido fue enviado con **ENVIAR**.
* PC servidor de impresión está encendido.
* **PUKA-HTTP está ejecutándose** en la barra de notificaciones.
* PC está conectada a la red correcta.
* IP del equipo no cambió.

---

## Pasos para verificar

1. Confirmar que el pedido fue enviado desde el POS con **ENVIAR**.
2. Revisar la **PC que funciona como servidor de impresión**.
3. Verificar que **PUKA-HTTP esté activo** en la barra de notificaciones de Windows.
4. Confirmar que la PC esté conectada a la red del restaurante (cable o Wi-Fi).
5. Verificar si el equipo cambió de red o de conexión (por ejemplo, de cable a Wi-Fi).
6. Si cambió la conexión, validar la **dirección IP actual del equipo**.
7. Si la IP cambió, actualizar la configuración de impresión o volver a asignar una **IP fija o reservada**.

---

## Resultado esperado

* La impresora de cocina vuelve a recibir las comandas.
* Los pedidos enviados desde el POS se imprimen correctamente.
* Se restablece la comunicación entre YURES y el servidor de impresión.

---

## Errores comunes

**Cambiar de cable a Wi-Fi**
→ La IP del equipo cambia y se pierde la conexión con la impresora.

**Cambiar de router o switch**
→ La red asigna una nueva IP al servidor de impresión.

**PUKA-HTTP no está ejecutándose**
→ El sistema no puede enviar la comanda a la impresora.

**PC servidor apagada o fuera de red**
→ Ninguna comanda llegará a cocina.
