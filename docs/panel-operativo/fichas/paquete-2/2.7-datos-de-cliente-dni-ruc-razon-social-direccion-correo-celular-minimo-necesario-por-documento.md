# 2.7 Datos de cliente

## DNI / RUC, Razón Social, Dirección, Correo, Celular

### Mínimo necesario por tipo de documento

No todos los documentos requieren lo mismo.
Ingresar datos innecesarios retrasa.
Ingresar datos incorrectos invalida el comprobante.

---

# 1️⃣ TICKET

## Datos obligatorios

Ninguno.

No requiere:

* DNI
* RUC
* Razón social
* Dirección

Es comprobante simple.

---

# 2️⃣ BOLETA ELECTRÓNICA

## Mínimo obligatorio

* DNI (si el cliente lo proporciona)
* Nombre (automático si consulta RENIEC)

En muchos casos:

* DNI puede ser opcional para montos pequeños.
* Pero si cliente lo solicita, debes ingresarlo.

## No obligatorio

* Dirección
* Correo
* Celular

---

# 3️⃣ FACTURA ELECTRÓNICA

Aquí no hay flexibilidad.

## Obligatorio

* RUC válido (11 dígitos)
* Razón social correcta
* Dirección fiscal

Sin eso:
No se debe emitir.

---

## Proceso correcto para Factura

1. Seleccionar **Factura Electrónica**.
2. Ingresar RUC.
3. Presionar botón de consulta.
4. Verificar:

   * Razón social
   * Dirección
5. Confirmar datos.
6. Luego proceder al pago.

---

# Campo Correo electrónico

No es obligatorio para emitir.

Se usa para:

* Enviar comprobante.
* Confirmación digital.

Si cliente lo solicita, debe ingresarse correctamente.

---

# Campo Celular / WhatsApp

No es requisito tributario.

Se usa para:

* Envío digital.
* Seguimiento comercial.

---

# Error común 1

Emitir factura con RUC mal digitado.

Consecuencia:

* Rechazo SUNAT.
* Documento inválido.
* Necesita anulación y nueva emisión.

---

# Error común 2

No consultar RUC y escribir razón social manual.

Consecuencia:

* Inconsistencia tributaria.
* Observación posterior.

---

# Error común 3

Emitir boleta cuando cliente necesita factura.

No se puede convertir después sin anulación.

---

# Regla operativa simple

Ticket → sin datos.
Boleta → DNI si cliente lo pide.
Factura → RUC validado obligatoriamente.

Nunca asumas.
Pregunta antes de cobrar.
