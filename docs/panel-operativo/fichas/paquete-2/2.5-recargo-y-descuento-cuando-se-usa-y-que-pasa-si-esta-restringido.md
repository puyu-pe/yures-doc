# 2.5 Recargo y Descuento

## Cuándo se usa y qué pasa si está restringido

Estas opciones modifican el total antes del pago.
No son formas de pago.

---

# RECARGO

## ¿Cuándo se usa?

Se aplica cuando:

* Se cobra comisión por POS.
* Se agrega servicio adicional.
* Se aplica un ajuste autorizado.

Ejemplo:
Total: 137.00
Recargo: 3.50
Nuevo total: 140.50

---

## Cómo aplicarlo

1. Presionar **RECARGO**.
2. Ingresar el monto.
3. Presionar **Aceptar**.

El total se actualiza automáticamente.

---

# DESCUENTO

## ¿Cuándo se usa?

Se aplica cuando:

* Hay promoción.
* Cliente frecuente.
* Error en precio.
* Autorización administrativa.

Ejemplo:
Total: 137.00
Descuento: 10.00
Nuevo total: 127.00

---

# Punto crítico

Recargo y Descuento modifican el TOTAL de la venta.

No se aplican después del pago.

Primero se ajusta.
Luego se cobra.

---

# Qué pasa si está restringido

En algunos locales:

* Solo supervisor puede aplicar descuento.
* Hay límite máximo permitido.
* Puede requerir clave.
* Puede estar bloqueado completamente.

Si está restringido:

* El sistema no permitirá aceptar.
* O solicitará autorización.
* O no mostrará la opción.

---

# Error común

Aplicar descuento después de registrar pago.

Resultado:

* Descuadre.
* Pago mayor al nuevo total.
* Inconsistencia en reportes.

---

# Error grave

Usar descuento para cuadrar caja.

Eso es incorrecto.

Descuento es decisión comercial, no herramienta para corregir errores de cobro.

---

# Regla operativa

1. Ajustar (recargo o descuento).
2. Verificar nuevo total.
3. Registrar pago.
4. Guardar.

El orden importa.
