# 2.6 Tipo de documento

**Ticket / Boleta electrónica / Factura electrónica — cuándo usar cada uno**

## Para qué sirve

Elegir el tipo correcto de comprobante al momento de **liquidar una venta**, según lo que solicite el cliente.

---

## Checklist rápido

* Cliente **no solicita comprobante tributario** → Ticket
* Cliente **persona natural con DNI** → Boleta electrónica
* Cliente **empresa con RUC** → Factura electrónica

---

## Cuándo usar cada uno

**Ticket**

→ Venta rápida donde el cliente **no solicita comprobante electrónico**.
→ Usado comúnmente en consumo inmediato o ventas rápidas.

---

**Boleta electrónica**

→ Cliente **persona natural**.
→ Puede solicitar registrar **DNI** o datos básicos.

---

**Factura electrónica**

→ Cliente **empresa o negocio**.
→ Requiere **RUC** y **razón social**.

---

## Pasos

1. Presionar **Liquidar venta**.
2. Seleccionar el tipo de documento:

   * **Ticket**
   * **Boleta electrónica**
   * **Factura electrónica**
3. Si es **Boleta o Factura**, ingresar los datos del cliente.
4. Registrar el medio de pago.
5. Presionar **Registrar Pago**.
6. Presionar **Guardar**.

---

## Resultado esperado

* Se genera el comprobante correspondiente.
* El documento se registra en el sistema.
* Se imprime el comprobante si corresponde.

---

## Errores comunes

**Cliente pidió factura pero se emitió ticket**
→ No se seleccionó **Factura electrónica** antes de guardar.

**No permite emitir factura**
→ Falta ingresar **RUC válido**.

**Datos incompletos del cliente**
→ El sistema puede requerir completar campos obligatorios.

**Se eligió boleta cuando era empresa**
→ Debe anularse y emitir nuevamente con **Factura electrónica**.
