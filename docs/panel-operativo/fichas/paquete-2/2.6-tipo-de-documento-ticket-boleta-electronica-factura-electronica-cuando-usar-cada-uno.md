# 2.6 Tipo de documento

## Ticket / Boleta electrónica / Factura electrónica

### Cuándo usar cada uno

Elegir mal el documento no es un detalle.
Tiene impacto tributario.

---

# 1️⃣ TICKET

## Cuándo usarlo

* Cliente no solicita comprobante con datos.
* Venta rápida.
* Consumo inmediato.
* No requiere DNI ni RUC.

Es un comprobante interno.

No se envía a SUNAT.

---

## Qué implica

* No permite crédito fiscal.
* No identifica al cliente.
* Solo respalda la venta en el sistema.

---

# 2️⃣ BOLETA ELECTRÓNICA

## Cuándo usarla

* Cliente es persona natural.
* Entrega DNI.
* No necesita crédito fiscal.
* Venta a consumidor final.

Se emite electrónicamente.

Se envía a SUNAT.

---

## Requisitos

* DNI (si lo solicita).
* Nombre.
* Puede llevar correo electrónico.

---

# 3️⃣ FACTURA ELECTRÓNICA

## Cuándo usarla

* Cliente es empresa.
* Solicita crédito fiscal.
* Entrega RUC.

Es obligatoria para empresas.

---

## Requisitos

* RUC válido.
* Razón social correcta.
* Dirección fiscal.

Si el RUC es incorrecto, la factura puede ser rechazada.

---

# Error común 1

Emitir Ticket cuando cliente pidió factura.

Resultado:

* Se debe anular.
* Genera proceso administrativo adicional.

---

# Error común 2

Emitir Boleta cuando cliente necesitaba Factura.

Resultado:

* Cliente no podrá usar crédito fiscal.
* Reclamo posterior.

---

# Error común 3

Emitir Factura sin validar RUC.

Resultado:

* Rechazo por SUNAT.
* Documento inválido.

---

# Regla operativa

Antes de registrar pago, confirmar:

“¿Requiere boleta o factura?”

No asumir.

---

# Resumen práctico

| Cliente           | Documento |
| ----------------- | --------- |
| No solicita datos | Ticket    |
| Persona natural   | Boleta    |
| Empresa con RUC   | Factura   |
