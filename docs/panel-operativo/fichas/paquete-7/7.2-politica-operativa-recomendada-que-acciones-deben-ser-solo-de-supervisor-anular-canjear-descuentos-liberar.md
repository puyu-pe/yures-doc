# 7.2 Política operativa recomendada: qué acciones deben ser solo de Supervisor

## Para qué sirve

Definir qué operaciones deben realizarse únicamente con **usuario Supervisor**, principalmente relacionadas con **control de caja y supervisión de la operación**.

---

## Checklist rápido

* Usuario operativo trabajando en **Punto de venta**.
* Supervisor disponible para control de caja cuando sea necesario.
* Acciones sensibles registradas con usuario Supervisor.

---

## Acciones que realiza el Supervisor en YURES

El modo Supervisor se utiliza principalmente para **control y ajustes de caja**, por ejemplo:

* **Abrir caja aunque ya se haya realizado el cierre.**
* Registrar **ingresos adicionales** después del cierre de caja.
* Registrar **egresos adicionales** después del cierre de caja.
* Consultar el **reporte de supervisor** para revisar la operación.

Estas acciones permiten corregir situaciones operativas como ingresos o gastos registrados después del cierre.

---

## Cómo funciona en la operación

1. El Supervisor ingresa al sistema en modo **Supervisor**.
2. Accede a las funciones de **caja o control**.
3. Realiza los ajustes necesarios (ingreso o egreso adicional).
4. La operación queda registrada con el usuario Supervisor.

---

## Resultado esperado

* Los ajustes posteriores al cierre quedan correctamente registrados.
* La caja mantiene trazabilidad de quién realizó el ajuste.
* El Supervisor puede revisar la operación mediante su reporte.

---

## Errores comunes

**Intentar registrar ingresos o egresos después del cierre con usuario normal**
→ El sistema no permitirá la operación.

**No registrar ajustes posteriores al cierre**
→ Puede generar descuadres en los reportes de caja.

**Usar siempre Supervisor para operar**
→ Se pierde trazabilidad del usuario operativo.
