# 7.3 Resolver “mesa pegada / pedido en dos mesas”

## Para qué sirve

Corregir situaciones donde un pedido aparece en **dos mesas** debido al uso de **Mover mesa** o **Juntar mesa**.

---

## Cuándo ocurre

Este problema suele aparecer cuando:

* Se usa **Mover a mesa**.
* Se usa **Juntar mesa** para unir mesas.
* Se intenta reorganizar mesas durante la atención.

El resultado es que el pedido puede verse en **más de una mesa**.

---

## Checklist rápido

* Verificar qué mesas están involucradas.
* Revisar si las mesas fueron **juntadas**.
* Confirmar qué mesa debe quedar con el pedido.

---

## Pasos

1. Identificar las mesas donde aparece el pedido.
2. Revisar cuál es la **mesa principal** del pedido.
3. Separar las mesas si fueron **juntadas**.
4. Verificar que el pedido quede solo en la mesa correcta.
5. Confirmar que las otras mesas queden **libres o disponibles**.

---

## Resultado esperado

* El pedido queda asociado a **una sola mesa**.
* Las demás mesas vuelven a estado **disponible**.
* No se pierde el consumo registrado.

---

## Errores comunes

**Intentar borrar ítems del pedido para corregirlo**
→ Puede generar errores en el consumo registrado.

**Liberar una mesa sin revisar el pedido**
→ Se puede perder la referencia del pedido.

**Mover mesas varias veces seguidas**
→ Puede confundir qué mesa contiene el pedido real.
