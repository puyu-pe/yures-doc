# 0.4 Botones base (barra inferior): CAJA / BÚSQUEDA VENTA / EGRESOS-INGRESOS / LÍMITE

## Para qué sirve

Acceder rápidamente a funciones clave sin salir del POS.

---

## Checklist rápido

* Saber qué acción necesita realizar.
* No confundir corrección de venta con movimiento de caja.
* Validar permisos si requiere autorización.

---

## Qué hace cada botón

### CAJA

→ Gestiona apertura, movimientos y control de caja del turno.
→ Se usa para revisar estado de caja o realizar operaciones relacionadas.

---

### BÚSQUEDA VENTA

→ Permite localizar ventas ya registradas.
→ Sirve para reimprimir, canjear o anular (según permisos).

---

### EGRESOS-INGRESOS

→ Registra movimientos de dinero que no son ventas.
Ejemplos:

* Pago de servicios.
* Compra menor.
* Ingreso adicional a caja.

No afecta ventas, pero sí el saldo de caja.

---

### LÍMITE

→ Controla disponibilidad de productos.
→ Permite ajustar cantidades máximas vendibles del día.

Evita vender productos sin stock disponible.

---

## Pasos (uso general)

1. Identificar la acción necesaria.
2. Presionar el botón correspondiente.
3. Ejecutar la operación.
4. Confirmar que la acción se registró correctamente.

---

## Resultado esperado

* Acción ejecutada sin afectar otras áreas.
* Caja y ventas permanecen coherentes.
* No se generan descuadres innecesarios.

---

## Errores comunes

**Registrar gasto como anulación de venta**
→ Debe usarse EGRESOS-INGRESOS, no BÚSQUEDA VENTA.

**Buscar una venta desde CAJA**
→ La consulta se realiza desde BÚSQUEDA VENTA.

**No ajustar LÍMITE cuando se acaba un producto**
→ Se siguen vendiendo productos sin disponibilidad real.

**Confundir movimiento de caja con venta**
→ Los egresos/ingresos no generan comprobante de venta.
