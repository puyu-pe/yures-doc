# 0.3 Canales rápidos: CALLE y DELIVERY (qué cambia y cuándo usarlo)

## Para qué sirve

Registrar ventas que no usan mesa física, diferenciando venta rápida en mostrador y pedidos para entrega.

---

## Checklist rápido

* Canal correcto seleccionado.
* Tipo de cliente identificado.
* Medio de pago claro.
* No confundir con venta en mesa.

---

## Qué cambia en cada canal

### CALLE

→ Venta rápida en mostrador.
→ No usa croquis ni mesa.
→ Flujo directo: pedido → liquidación.

### DELIVERY

→ Pedido para envío a domicilio.
→ Puede requerir datos del cliente y dirección.
→ Se gestiona entrega desde el módulo correspondiente.

---

## Cuándo usar cada uno

**Usar CALLE cuando:**

* Cliente paga en mostrador.
* No se asigna mesa.
* Es venta inmediata.

**Usar DELIVERY cuando:**

* Pedido se envía fuera del local.
* Requiere control de entrega.
* Se debe hacer seguimiento del despacho.

---

## Pasos

1. Desde la vista principal, presionar **CALLE** o **DELIVERY**.
2. Verificar que el sistema cambie al modo correspondiente.
3. Registrar pedido según el flujo del canal.
4. Liquidar y cobrar.

---

## Resultado esperado

* Venta queda registrada en el canal correcto.
* No se ocupa una mesa innecesariamente.
* Reportes reflejan correctamente el tipo de venta.

---

## Errores comunes

**Registrar venta de mostrador como mesa**
→ Afecta ocupación y control del salón.

**Usar CALLE para pedido a domicilio**
→ No permite seguimiento de entrega.

**Olvidar cambiar de canal**
→ La venta se registra en el canal anterior.

**No validar que el canal quedó activo**
→ Puede continuar trabajando en modo mesa sin notarlo.
