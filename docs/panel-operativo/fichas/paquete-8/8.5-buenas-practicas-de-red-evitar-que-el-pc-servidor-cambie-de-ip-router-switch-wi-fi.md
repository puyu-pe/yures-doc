# 8.5 Buenas prácticas de red: evitar que el PC servidor cambie de IP (router/switch/Wi-Fi)

## Para qué sirve

Evitar que la impresión de comandas o tickets falle cuando la PC que actúa como **servidor de impresión (PUKA-HTTP)** cambia de dirección IP.

---

## Checklist rápido

* PC servidor **encendida**.
* **PUKA-HTTP activo** en la barra de notificaciones.
* PC conectada **siempre a la misma red**.
* IP del servidor **no cambia**.

---

## Buenas prácticas recomendadas

**Usar cable de red (LAN)**
→ Evita cambios de IP frecuentes que ocurren en Wi-Fi.

**Asignar IP fija o reservada en el router**
→ Garantiza que el servidor de impresión siempre tenga la misma dirección.

**No cambiar de red (LAN ↔ Wi-Fi)**
→ Cambiar el tipo de conexión suele generar una nueva IP.

**No mover la PC a otro switch/router sin revisar la red**
→ Puede quedar en otro segmento de red y perder comunicación.

---

## Resultado esperado

* Las impresoras reciben comandas correctamente.
* No se pierde comunicación con cocina.
* PUKA-HTTP funciona de forma estable durante toda la operación.

---

## Errores comunes

**Las comandas dejaron de imprimirse de repente**
→ La PC servidor cambió de IP.

**Se cambió el cable a Wi-Fi y dejó de imprimir**
→ La red asignó una IP diferente.

**Se cambió router o switch**
→ Puede requerir revisar configuración de red y PUKA-HTTP.

**PUKA-HTTP no aparece activo**
→ El servicio no está ejecutándose en la PC servidor.
